
# 通用代码任务 Prompt 模板

以下模板可复用于大多数编码场景，按需删减章节即可。

## 模板

```markdown
## 背景
[简述业务背景与当前问题]

## 目标
[期望 AI 完成的具体任务，一条一句]

## 约束
- 技术栈：[语言 / 框架 / 版本]
- 不要修改：[明确排除的文件或模块]
- 遵循：[项目内已有的命名 / 风格约定]

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 参考
- 相关文件：`path/to/file.ts`
- 类似实现：`path/to/example.ts`
```

## 使用技巧

1. **背景与目标分离**：避免 AI 在缺少上下文时猜测意图
2. **约束写清楚**：可显著减少无关改动
3. **给出参考文件**：比纯文字描述更高效
4. **验收标准可测试**：便于 Agent 自检

## 反例

❌ 「帮我优化一下这段代码」—— 过于模糊

✅ 「重构 `UserService.getProfile`，提取重复的数据库查询逻辑，保持对外 API 不变，并补充单元测试」
