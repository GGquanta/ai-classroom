# 如何使用 AI 工具快速整理会议纪要

## 写在前面

会议结束后，往往留下几页潦草笔记、一叠扫描件，或手机里几十张幻灯片照片。要把这些材料变成一份**结构清晰、可追溯、能传阅**的会议纪要，手工整理通常既费时又容易遗漏关键信息。

人工智能工具（大语言模型、视觉识别模型等）可以在这一环节显著提速：帮你补全句子、理顺结构、识别图片里的文字，甚至按固定模板输出成稿。但 AI 不是「一键代写」——**你提供什么素材、下什么指令、事后如何校对**，决定了纪要是否准确、可用。

本文是一份面向普通职场人的**实操教程**，介绍几种常见整理路径、应当记录哪些核心信息，以及如何与 AI 协作。文中示例均为虚构场景，不涉及任何真实单位或会议。

---

## 一、会议纪要应当记录什么

无论采用哪种整理方式，一份合格的纪要通常应覆盖以下要素。建议在会议进行时就尽量记全，事后整理会轻松很多。

### 1.1 基本信息（「谁、何时、何地、为何」）

| 要素 | 说明 | 记录建议 |
|------|------|----------|
| 会议主题 | 本次会议讨论什么 | 用一句话概括，避免过于笼统 |
| 时间 | 日期与起止时段 | 精确到日期；若有跨时区参会，注明时区 |
| 地点 | 线下会议室或线上平台 | 线下写清楼宇与房间号；线上写平台名称 |
| 主持人 | 谁主持会议 | 姓名 + 单位/部门 |
| 记录人 | 谁负责整理纪要 | 便于后续核对与修订 |

### 1.2 参会方与人物

- **列出主要参会单位**，各方用分段或列表区分。
- **尽量写全姓名与职务**（或角色，如「产品负责人」「法务顾问」）。
- 若仅记了姓氏或缩写（如「张」「李总」），整理阶段应结合签到表、名片或会议邀请补全；无法确认的，标注「待核实」。

### 1.3 单位背景与交流主题

纪要读者未必熟悉每家单位的业务。建议在正文开始前或各方首次发言处，用一两句话说明：

- 对方单位主要从事什么业务；
- 本次会议与己方关注的**具体议题**是什么（如产品对接、合作模式、技术方案评审等）。

背景不必写成企业宣传稿，**够用即可**，目的是让一个月后重读纪要的人仍能看懂语境。

### 1.4 对话内容：必须标注讲话人

这是许多初稿最容易出问题的地方：**把各方观点混成一段「会议认为」**，事后无法分清谁承诺了什么。

正确做法是：

- 按**发言顺序**或**议题模块**组织正文；
- **每一段观点或答复都标明讲话人**（姓名 + 单位），例如：

> **王明（甲公司市场部）**介绍了本年度渠道拓展计划，并提出希望在下季度开展联合试点。  
> **李芳（乙公司技术部）**回应说，试点需先完成接口对接测试，预计需要四周。

![规范纪要写法示意：左侧为未标注讲话人的混乱段落，右侧为按「姓名（单位）」分段组织的正文](images/meeting-minutes-speaker-attribution.png)

对于「问答」类交流，建议用「提问要点 / 答复结论」分栏或分小段呈现，避免问题和答案散落在不同章节。

### 1.5 共识、待办与补充说明

- **已达成一致的事项**：用条目列出，表述尽量可执行。
- **待办事项**：谁、做什么、何时完成（若会议中已约定）。
- **补充说明**：数据口径、存疑表述、尚需书面确认的内容等，单独说明，避免与定论混在一起。

可在文末加一句非约束性声明，例如：「本文根据会议发言整理，具体权利义务以双方后续签署的正式文件为准。」

## 二、四种常见的 AI 辅助整理方法

不同素材适合不同路径。下面四种方法可以单独使用，也可以组合：**先识别成文字，再交给 AI 结构化**。

![AI 辅助整理会议纪要流程示意：手写笔记、幻灯片照片、扫描件等素材经识别与结构化，输出规范纪要文档](images/ai-meeting-minutes-workflow.png)

```
素材类型          典型路径
─────────────────────────────────────────
要点式笔记    →  关键词扩充 + 结构化
扫描件/手写稿 →  OCR 识别 → 校对 → 结构化
幻灯片照片    →  视觉识别 → 与笔记合并 → 结构化
录音/速记稿   →  转写 → 分段归因 → 结构化
```

### 2.1 方法一：关键词扩充（要点笔记 → 完整纪要）

**适用场景**：你在会议中记了提纲式要点——几个词、半句话、数字和专有名词——希望扩写成通顺、完整的纪要段落。

**操作步骤**：

1. **保留原始要点**，不要先自己大改，把笔记原样提供给 AI。
2. **同时提供上下文**：会议主题、参会方、已知的人员名单。
3. **明确指令**，例如：

   > 请根据以下会议要点，整理为规范的会议纪要正文。要求：  
   > 1）按实际交流顺序组织，不要过度概括；  
   > 2）为每段观点补充讲话人姓名（见参会名单）；  
   > 3）将短语扩写为完整句子，但不改变原意；  
   > 4）数字、结论、承诺表述保持与笔记一致。

4. **人工通读**：重点核对数字、日期、是否把「讨论」写成了「决定」。

**优点**：速度快，适合数字化笔记、手机备忘录。  
**注意**：AI 可能「脑补」未讨论的内容；对不确定的要点，应标注「会上提及，细节待确认」。

---

### 2.2 方法二：OCR 识别（扫描 PDF、手写笔记）

**适用场景**：纸笔记录后扫描成 PDF，或收到纸质会议材料的电子版扫描件。

**推荐分两阶段处理**，不要指望一步得到终稿。

#### 阶段 A：视觉识别 → 逐页原文

使用支持「看图识字」的 AI 或 OCR 服务，将每一页转为文字。建议：

- 指定输出为**纯文本或 Markdown**，按页分段，便于对照原图检查；
- 在指令中要求：**逐行转写，不要概括、不要省略**；
- 若纸张有**水印、印章、页眉装饰**，明确要求忽略，避免误入正文；
- 手写稿识别率低于印刷体，字迹潦草处模型可能用「□」或空白标出，需人工补全。

**常见问题与处理**：

| 现象 | 处理建议 |
|------|----------|
| 个别页内容缺失 | 单独重识该页，或提高图片清晰度后重试 |
| 误把水印当正文 | 在指令中列举需忽略的字样；后处理时删除重复水印行 |
| 公式、表格乱码 | 若页面无公式，要求禁用特殊排版符号，改用 plain text |
| 跨页句子断开 | 第二阶段合并时按语义衔接 |

#### 阶段 B：OCR 原文 → 规范纪要

将阶段 A 的输出交给文本类 AI，指令示例：

> 以下是手写会议笔记的 OCR 结果，可能存在错字和断行问题。请：  
> 1）修正明显识别错误，理顺标点和段落；  
> 2）按交流顺序保留原始结构，不要大幅压缩；  
> 3）根据上下文补全讲话人姓名；  
> 4）输出为包含「基本信息、背景、主要内容、后续安排」的会议纪要。

**优点**：解放大量录入时间。  
**注意**：OCR + AI 润色是两道误差来源，**务必对照扫描件抽查至少 30% 内容**。

---

### 2.3 方法三：幻灯片照片识别

**适用场景**：演讲者未提供 PPT 源文件，你用手机拍摄了投影画面，或会议资料以图片形式分发。

**操作步骤**：

1. **按页码或时间顺序**整理照片，删除模糊、重复拍摄的帧。
2. 使用视觉模型**逐张识别**幻灯片上的标题、正文、图表标注文字。
3. 将识别结果与**你自己的听讲笔记**合并：幻灯片提供「讲了什么框架」，笔记补充「现场讨论、问答、偏离讲稿的补充意见」。
4. 交给 AI 时说明分工，例如：

   > 附件 1 为演讲幻灯片 OCR 文本，附件 2 为我记录的讨论要点。请合并为纪要：幻灯片内容用于归纳主讲部分，讨论要点用于补充问答与各方发言，并标注讲话人。

**优点**：快速还原演示脉络，适合培训、路演、汇报类会议。  
**注意**：图表中的数据务必与照片核对；动画、口头举例不会出现在幻灯片上，不能仅靠识图完成纪要。

---

### 2.4 方法四：录音转写 + 发言人区分（补充路径）

**适用场景**：会议有录音或实时转写稿。

流程简述：音频 → 语音转文字 →（可选）说话人分离 → AI 按议题分段 → 人工校对专有名词。

与手写 OCR 类似，**转写稿宜先「忠实记录」再「整理格式」**。若工具无法自动区分说话人，应在转写稿中根据声线、称呼或记忆手工标注「发言人 A / B」，再请 AI 对照参会名单替换为真实姓名。

## 三、推荐的标准文档结构

为便于传阅与归档，建议将终稿整理为**五个部分**（可根据会议类型微调标题）：

1. **会议基本信息**——主题、时间、地点、主持人、记录人、参会单位与名单（表格呈现更清晰）。
2. **会议背景与目的**——为什么开这场会、期望达成什么。
3. **会议主要内容**——按议题或发言顺序展开，**段段有主语、有讲话人**。
4. **共识与后续工作**——已达成的一致意见、分工与时间节点。
5. **补充说明**——存疑内容、数据口径、免责声明等。

文件命名可采用统一规则，便于检索，例如：

`会议纪要-年份-月份-日期-主题关键词.pdf`

## 四、与 AI 协作的实用技巧

### 4.1 把指令写具体，而不是「帮我写纪要」

低效指令：

> 帮我整理会议纪要。

更高效指令应包含：

- **素材类型**（要点 / OCR 原文 / 幻灯片文字 / 转写稿）；
- **结构要求**（五段式、是否要表格）；
- **忠实度要求**（不得改动数字与结论、不得过度概括）；
- **人物信息**（参会名单或能确定的姓名职务）；
- **输出格式**（Markdown、Word 大纲、纯文本）。

### 4.2 分步完成，优于一次生成终稿

典型工作流：

```
原始素材 → 识别/转写（若有） → 纠错与补全发言人 → 结构化排版 → 人工终审
```

每一步都可以单独让 AI 执行，你在中间插入校对，整体准确率高于「从照片直接出终稿」。

### 4.3 提供「范例格式」而非长篇说教

若你或团队已有写得较好的纪要，可脱敏后作为**格式范例**发给 AI：

> 请参照下列范例的章节划分、表格样式和发言人标注方式，整理本次会议内容（范例仅作版式参考，勿复制其中任何业务信息）。

### 4.4 控制「归纳程度」

- **内部存档、争议复盘**：宜保留较多原话结构，归纳程度低。
- **对外简报、高层摘要**：可在忠实版基础上另做一版「一页纸摘要」，并标明与完整纪要的关系。

切忌让 AI 把尚未形成决议的讨论写成「双方决定……」。

### 4.5 敏感信息先脱敏，再送入云端工具

涉及未公开财务数据、个人身份信息、合同条款时：

- 本地记录完整版；
- 送入公有云 AI 前做脱敏，或选用合规的企业版 / 本地部署工具；
- 对外发布的教程、案例应使用虚构数据。

## 五、质量自检清单

成稿前建议逐项勾选：

**内容与忠实度**

- [ ] 时间、地点、单位名称无误
- [ ] 每位主要发言人姓名、职务正确
- [ ] 对话段落均标明讲话人，无「无主语」大段论述
- [ ] 数字、日期、比例与原始素材一致
- [ ] 未把「探讨」写成「已决定」

**结构与可读性**

- [ ] 章节完整，标题层级清晰
- [ ] 背景说明足够让「未参会者」读懂
- [ ] 待办事项具备「谁、做什么、何时」要素（若会议已约定）
- [ ] 存疑处已标注，未混入定论

**识别类素材专项（OCR / 幻灯片）**

- [ ] 各页内容无大面积缺失
- [ ] 水印、页眉装饰未误入正文
- [ ] 专有名词、人名已人工核对
- [ ] 跨页断句已理顺

## 六、虚构示例：从要点笔记到纪要片段

**原始笔记（虚构）**：

```
3.15 下午 科技园 A 栋 301
产品对接会 — 甲公司 / 乙科技
张、刘、陈；乙方：赵工、孙经理

张：介绍新版 API，Q2 上线
赵：关心并发上限、文档
孙：希望联合白皮书，六月前？
刘：法务看过草案，还改了两处
陈：问试点城市 — 赵：先北上广深，再扩
待办：乙发测试账号；甲两周内回白皮书意见
```

**经 AI 扩充后的纪要片段（虚构，仅示范写法）**：

> **张明（甲公司产品部）**介绍了新版开放接口的规划，表示计划于第二季度正式上线。  
> **赵工（乙科技研发部）**询问高并发场景下的性能上限及技术文档发布时间。  
> **孙经理（乙科技市场部）**提议双方联合发布行业白皮书，并希望能在六月前完成初稿。  
> **刘洋（甲公司法务部）**说明已审阅合作草案，目前仍有待修改条款。  
> **陈晨（甲公司商务部）**询问试点落地城市范围。**赵工**答复拟优先在一线城市开展试点，后续再逐步扩展。

可见：要点未改义，但已具备**人物、单位、完整句**，可直接进入正式纪要正文。

## 七、小结：人机分工建议

| 环节 | 建议由人完成 | 建议由 AI 辅助 |
|------|--------------|----------------|
| 会中记录 | 关键数字、决议、发言人 | — |
| 素材整理 | 选图、排序、脱敏 | 批量 OCR、转写 |
| 初稿撰写 | 核对事实 | 扩写要点、排版结构 |
| 终审发布 | 法律责任、对外口径 | 润色语言、统一格式 |

AI 工具擅长**提速与结构化**；**真实性、可归因性、合规性**仍须由记录人与参会方共同把关。掌握「记什么、怎么分步指令、如何校对」之后，大多数常规会议纪要的整理时间可以从数小时缩短到数十分钟——前提是你在会议当下就为「事后可追溯」多记一笔：尤其是**谁说了什么**。
